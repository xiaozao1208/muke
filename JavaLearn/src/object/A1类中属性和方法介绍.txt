java  面向对象的编程思想
    解决问题的时候按照现实生活中的规律来考虑问题
    考虑在解决问题的过程中，有几个实体参与进来
    理解为： 实体动作动作的支配者，没有实体动作就发生不了。

    人 冰箱 大象
    分析每一类 个体都有神马特点 做了哪些事情
    大象 特点
    人 特点
    冰箱 特点

    在现实生活中， 对象是先有的，总结出来的概念（类）后产生的
    在计算机中录用面向对象的思想来做事情：
        需要先自己定义（描述）一个类（型）；
        想要做事情 需要在描述的类中创建一个具体的个出来个体（对象）来做事

    类和对象
    类-人类 抽象笼统的概念 描述一类事物 肯定是具有相同的特征行为
            人类有名字，年龄，性别————静态描述特征----属性
            人类能吃饭，能说话，能学习-动态动作行为----方法
    对象-具体的人   人类中的一个具体的人，比如：李世民
            李世民这个具体的人，有名字，性别，年龄
                             能吃饭，能说话，能学习

    如何在计算机中创建/描述一个类：
    1、先创建一个类 class
    2、类的里面 利用属性或方法去描述这个类
    3、创建一个当前类的对象，让对象调用属性/方法做事

类中的方法-----做一件事情（*** 很重要）
    权限修饰符  [特征修饰符]  返回值类型  方法名字(参数列表) [抛出异常][{
        方法体
    }]

    必须有的结构：
    权限修饰符  返回值类型 方法名字(参数列表){
    }

    1.无参数 无返回值
        public void test(){}
    2.无参数 有返回值
		
    3.有参数 无返回值
    4.有参数 有返回值
	
	可以理解为
		方法 是做了一件事情
		返回值 是这件事情做完了,必须留下一个结果
		参数是做事情之前,必须提供的条件;
